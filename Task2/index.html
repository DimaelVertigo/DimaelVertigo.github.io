Мы ожидаем от request результаты /countries, /cities и /populations, но получаем только от последнего /populations. Мы воспользовались механизмом оборачивания в анонимную самовызывающую функцию.
Три раза создали три разные области. В этих областях вызывается три разных callback, каждый из которых захочет свой request. Для это он отправится на внешнюю область видимости, т.к. у него внутри не объявлена переменная request, и будет в своей изолированной области. У каждой будет свой request. Три наши области были объявлены синхронно. Значит, каждый раз, когда происходил цикл, он получает свой request с правильным значением. Второй раз - создается новая область и получает свое значение и не перезаписывает предыдущее. Когда сработают три callback, у нас будет три разных callback в разных областях видимости и соответственно свой request.
